1. How to Download Git on Mac/Windows/Linux

Installing Git on Windows

  1. Go to https://git-scm.com/download/win
  2. Download the latest Git for Windows installer
  3. Run the installer and follow the installation wizard
  4. Choose the default options unless you have specific preferences
  5. Complete the installation

Installing Git on macOS

There are several ways to install Git on macOS:

Option 1: Using Homebrew (recommended) 

# Install Homebrew if you don't have it
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Git
brew install git

Option 2: Download the installer

  1. Go to https://git-scm.com/download/mac
  2. Download and run the macOS Git installer

Installing Git on Linux

For Debian/Ubuntu based distributions: 

sudo apt update
sudo apt install git

For Fedora:

sudo dnf install git

For CentOS/RHEL:

sudo yum install git

Verification Tip

After installation, verify Git is properly installed by opening a terminal or command prompt and typing: 

git --version

You should see the Git version number displayed.

2. GitHub Pages & CI/CD: Website Deployment

GitHub Pages & CI/CD Mastery: Automate Your Website Deployment

GitHub Pages is a static site hosting service that takes HTML, CSS, and JavaScript files directly from a GitHub repository and publishes a website.

Setting Up GitHub Pages

  1. Create a GitHub account at github.com if you don't have one already
  2. Create a new repository named username.github.io, where username is your GitHub username
  3. Clone the repository to your local machine
  4. Add your website files (HTML, CSS, JavaScript) to the repository
  5. Commit and push the changes to GitHub
  6. Wait a few minutes, then visit username.github.io to see your website live!

Introduction to CI/CD with GitHub Actions

Continuous Integration and Continuous Deployment (CI/CD) is a method to frequently deliver apps by introducing automation into the development process.

GitHub Actions allows you to automate your workflow from idea to production:

  1. Create a .github/workflows directory in your repository
  2. Add a YAML file (e.g., deploy.yml) to define your workflow
  3. Define the workflow to build and deploy your website automatically

Here's a basic GitHub Actions workflow example for deploying to GitHub Pages: 

name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v2

      - name: Deploy
        uses: JamesIves/github-pages-deploy-action@4.1.4
        with:
          branch: gh-pages
          folder: .

Pro Tip:

Using GitHub Actions, you can automate tasks like running tests, linting code, and deploying your application whenever you push changes to your repository.
